# Αρχείο Πράξεων Δήμου Γαύδου — από τη Διαύγεια

Ένα πλήρες σύστημα που κατεβάζει **όλες** τις επίσημες πράξεις (αποφάσεις,
ανακοινώσεις, διαγωνισμούς κ.λπ.) του Δήμου Γαύδου από το επίσημο ανοιχτό API
της Δι@ύγειας, τις αποθηκεύει σε βάση δεδομένων, **ανανεώνεται αυτόματα** με τις
νέες, και τις εμφανίζει σε μια ιδιωτική (κωδικός πρόσβασης) ιστοσελίδα.

Πηγή: Πρόγραμμα Δι@ύγεια (Υπουργείο Ψηφιακής Διακυβέρνησης). Τα δεδομένα
διατίθενται ελεύθερα με άδεια CC-BY. Δεν χρειάζεται API key.

---

## Τι περιλαμβάνει

| Αρχείο | Τι κάνει |
|---|---|
| `diavgeia_sync.py` | Κατεβάζει τις πράξεις. Πρώτη φορά → όλο το αρχείο. Μετά → μόνο τις νέες. |
| `app.py` | Ιδιωτική ιστοσελίδα (με κωδικό) για αναζήτηση/φιλτράρισμα/προβολή. |
| `export.py` | Εξαγωγή όλης της βάσης σε CSV και JSON. |
| `run_sync.sh` | Βοηθητικό script για αυτόματη εκτέλεση (cron). |
| `requirements.txt` | Οι βιβλιοθήκες Python που χρειάζονται. |

---

## Βήμα 1 — Εγκατάσταση (μία φορά)

Χρειάζεσαι Python 3.9+.

```bash
cd gavdos-diavgeia
pip install -r requirements.txt
```

## Βήμα 2 — Πρώτο «γέμισμα» της βάσης

Αυτό κατεβάζει **ολόκληρο** το ιστορικό αρχείο του Δήμου Γαύδου:

```bash
python3 diavgeia_sync.py --db gavdos.db --full
```

Θα δημιουργηθεί το αρχείο `gavdos.db` (βάση SQLite) με όλες τις πράξεις.
Στο τέλος θα σου πει πόσες αποθήκευσε.

> Σημείωση: αν το API της Διαύγειας είναι σε προγραμματισμένη συντήρηση
> (εμφανίζεται σχετικό μήνυμα στη σελίδα τους), απλώς ξανατρέξε το αργότερα —
> το script συνεχίζει από εκεί που έμεινε.

## Βήμα 3 — Αυτόματη ανανέωση

Κάθε επόμενη εκτέλεση κατεβάζει **μόνο τις νέες** πράξεις (γρήγορο):

```bash
python3 diavgeia_sync.py --db gavdos.db
```

### Να τρέχει μόνο του

**Linux / macOS (cron):** Άνοιξε το crontab με `crontab -e` και πρόσθεσε μία
γραμμή ώστε να τρέχει π.χ. κάθε μέρα στις 07:00:

```
0 7 * * * /πλήρης/διαδρομή/gavdos-diavgeia/run_sync.sh
```

(Διόρθωσε τη διαδρομή μέσα στο `run_sync.sh` ώστε να δείχνει στον φάκελο.)

**Windows (Task Scheduler):** Δημιούργησε νέα εργασία που εκτελεί
`python diavgeia_sync.py --db gavdos.db` στον φάκελο του project, με
χρονοδιάγραμμα της επιλογής σου (π.χ. καθημερινά).

## Βήμα 4 — Η ιδιωτική ιστοσελίδα

Όρισε όνομα χρήστη και έναν δυνατό κωδικό, και ξεκίνα τον server:

```bash
export VIEWER_USER=adam
export VIEWER_PASSWORD='βάλε-έναν-δυνατό-κωδικό'
export GAVDOS_DB=gavdos.db
python3 app.py
```

Άνοιξε στον browser: `http://localhost:8000`
Θα σου ζητήσει το όνομα χρήστη και τον κωδικό. Μέσα έχεις αναζήτηση (θέμα/ΑΔΑ),
φίλτρα ανά τύπο πράξης και έτος, και συνδέσμους στη Διαύγεια και στα έγγραφα.

(Στα Windows το `export` γίνεται `set`, π.χ. `set VIEWER_PASSWORD=...`.)

## Βήμα 5 — Εξαγωγή (προαιρετικά)

```bash
python3 export.py --db gavdos.db --out exports/
```

Παράγει `exports/gavdos_decisions.csv` (ανοίγει σε Excel) και `.json`.

## Πώς να το κάνεις online με ιδιωτική πρόσβαση

Έχεις μερικές καλές επιλογές, από την πιο απλή στην πιο «σοβαρή»:

1. **Δικός σου υπολογιστής + Cloudflare Tunnel / Tailscale** — Τρέχεις το
   `app.py` στο μηχάνημά σου και το εκθέτεις με ασφάλεια μόνο σε όσους θες.
   Φθηνό/δωρεάν, αλλά ο υπολογιστής πρέπει να είναι ανοιχτός.

2. **Μικρό VPS** (π.χ. Hetzner, DigitalOcean — ~4–6€/μήνα) — Ανεβάζεις τον
   φάκελο, βάζεις το `run_sync.sh` στο cron για την αυτόματη ανανέωση, και
   τρέχεις το `app.py` πίσω από έναν reverse proxy (nginx) με HTTPS. Αυτή είναι
   η πιο σταθερή λύση για 24/7 online βάση.

3. **Πλατφόρμες τύπου Railway / Render / Fly.io** — Ανεβάζεις τον κώδικα, ορίζεις
   τις μεταβλητές `VIEWER_USER` / `VIEWER_PASSWORD`, και προγραμματίζεις το sync
   ως cron job της πλατφόρμας.

Σε κάθε περίπτωση: η πρόσβαση προστατεύεται ήδη με κωδικό (HTTP Basic auth). Για
σοβαρή χρήση, βεβαιώσου ότι ο ιστότοπος τρέχει πάνω από **HTTPS** ώστε ο κωδικός
να ταξιδεύει κρυπτογραφημένος (όλες οι παραπάνω πλατφόρμες το δίνουν δωρεάν).

## Σημαντικά / περιορισμοί

- Το όνομα φορέα στη Διαύγεια είναι `gavdou_dimos` (ορίζεται στο
  `diavgeia_sync.py`, μεταβλητή `ORG_UID`). Αν κάποια στιγμή χρειαστεί να
  προσθέσεις και νομικά πρόσωπα/επιχειρήσεις του Δήμου, μπορείς να τρέξεις το
  sync με διαφορετικό `ORG_UID`.
- Το API επιστρέφει μεταδεδομένα + σύνδεσμο στο PDF κάθε πράξης. Τα ίδια τα PDF
  δεν κατεβαίνουν (για να μη φουσκώσει η βάση), αλλά το link αποθηκεύεται. Αν
  θες και τοπική αποθήκευση των PDF, μπορώ να το προσθέσω.
- Τα ονόματα πεδίων του API ενίοτε αλλάζουν ανά έκδοση· ο parser ψάχνει πολλαπλά
  πιθανά κλειδιά και κρατάει και το πλήρες `raw_json` κάθε πράξης, ώστε να μη
  χαθεί τίποτα ακόμα κι αν αλλάξει η δομή.
