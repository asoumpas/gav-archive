# Οδηγός βήμα-βήμα — Δωρεάν στο GitHub

Αυτός ο οδηγός σε πάει από το μηδέν μέχρι μια online βάση με όλες τις πράξεις
του Δήμου Γαύδου, που **ανανεώνεται μόνη της κάθε μέρα**, χωρίς να πληρώσεις
τίποτα και χωρίς να χρειάζεσαι server. Όλα γίνονται μέσα στο GitHub.

Δεν χρειάζεται να ξέρεις προγραμματισμό. Απλώς ακολούθησε τα βήματα με τη σειρά.

---

## Τι θα έχεις στο τέλος

- Μια ιστοσελίδα (σε «κρυφό» link) με αναζήτηση & φίλτρα σε όλες τις πράξεις.
- Αυτόματο κατέβασμα των νέων αποφάσεων κάθε μέρα.
- Όλα δωρεάν, για πάντα.

---

## ΒΗΜΑ 1 — Φτιάξε ένα νέο "repository" (αποθετήριο)

Σκέψου το repository σαν έναν φάκελο στο διαδίκτυο που κρατάει τα αρχεία σου.

1. Μπες στο https://github.com και κάνε login.
2. Πάνω δεξιά, πάτα το **+** → **New repository**.
3. Στο **Repository name** γράψε π.χ. `gavdos-diavgeia`.
4. **ΣΗΜΑΝΤΙΚΟ:** διάλεξε **Private** (ιδιωτικό). Έτσι τα αρχεία και ο κώδικας
   δεν φαίνονται δημόσια.
5. Μην τσεκάρεις τίποτα άλλο. Πάτα **Create repository**.

---

## ΒΗΜΑ 2 — Ανέβασε τα αρχεία

1. Στη σελίδα του νέου repository, πάτα τον σύνδεσμο **uploading an existing file**
   (ή: κουμπί **Add file** → **Upload files**).
2. Άνοιξε το zip που σου έδωσα (`gavdos-diavgeia.zip`) στον υπολογιστή σου και
   **σύρε ΟΛΑ τα αρχεία και τους φακέλους** μέσα στο παράθυρο του GitHub.

   > Πρόσεξε: πρέπει να ανέβει και ο **κρυφός φάκελος `.github`** (περιέχει την
   > αυτοματοποίηση). Αν στον υπολογιστή σου δεν τον βλέπεις, ενεργοποίησε την
   > εμφάνιση κρυφών αρχείων, ή σύρε όλον τον φάκελο `gavdos-diavgeia` μαζί.

3. Κάτω-κάτω πάτα **Commit changes** (πράσινο κουμπί). Περίμενε να ανέβουν.

---

## ΒΗΜΑ 3 — Επίτρεψε στην αυτοματοποίηση να γράφει

1. Στο repository, πάνω, πάτα **Settings**.
2. Αριστερά: **Actions** → **General**.
3. Κατέβα κάτω στο **Workflow permissions**.
4. Διάλεξε **Read and write permissions** και πάτα **Save**.

(Αυτό αφήνει το αυτόματο sync να αποθηκεύει τις νέες πράξεις πίσω στο repository.)

---

## ΒΗΜΑ 4 — Κατέβασε όλο το αρχείο (πρώτη φορά)

1. Πάνω, πάτα την καρτέλα **Actions**.
2. Αν σου ζητήσει, πάτα το πράσινο **I understand my workflows, enable them**.
3. Αριστερά, διάλεξε **First full download (run once)**.
4. Δεξιά, πάτα **Run workflow** → ξανά **Run workflow**.
5. Περίμενε. Θα δεις έναν κίτρινο κύκλο που γίνεται πράσινο ✓ όταν τελειώσει
   (μπορεί να πάρει μερικά λεπτά, ανάλογα με το πόσες πράξεις υπάρχουν).

Όταν γίνει πράσινο, όλη η βάση έχει κατέβει.

---

## ΒΗΜΑ 5 — Άναψε την ιστοσελίδα (GitHub Pages)

1. **Settings** → αριστερά **Pages**.
2. Στο **Source** διάλεξε **Deploy from a branch**.
3. Στο **Branch** διάλεξε `main`, και στον φάκελο διάλεξε **/docs**. Πάτα **Save**.
4. Περίμενε ~1 λεπτό και κάνε ανανέωση τη σελίδα. Θα εμφανιστεί το link, κάτι σαν:
   `https://to-onoma-sou.github.io/gavdos-diavgeia/`

Αυτό είναι το «κρυφό» link σου. Δούλεψε το, και δώσ' το μόνο σε όποιον θες.

> Σημείωση για το «ιδιωτικό»: επειδή το GitHub Pages στη δωρεάν έκδοση είναι
> τεχνικά προσβάσιμο, βάλαμε στη σελίδα οδηγία να **μην** την καταγράφουν οι
> μηχανές αναζήτησης (noindex), και το repository είναι Private ώστε να μη
> φαίνεται ο κώδικας. Πρακτικά, κανείς δεν θα τη βρει αν δεν του δώσεις το link.

---

## ΒΗΜΑ 6 — Τίποτα! Δουλεύει μόνο του

Από εδώ και πέρα, κάθε μέρα το GitHub θα τρέχει αυτόματα το «sync», θα τραβάει
τις νέες πράξεις και θα ενημερώνει τη σελίδα. Δεν χρειάζεται να κάνεις τίποτα.

Αν ποτέ θες να το «σπρώξεις» χειροκίνητα: **Actions** → **Sync Gavdos decisions**
→ **Run workflow**.

---

## Συχνές απορίες

**Δεν βλέπω τον φάκελο `.github` για να τον ανεβάσω.**
Είναι κρυφός. Στα Windows: στον File Explorer → View → τσέκαρε "Hidden items".
Στο Mac: πάτα `Cmd + Shift + .` μέσα στο Finder. Εναλλακτικά ανέβασε ολόκληρο
τον φάκελο `gavdos-diavgeia` με drag & drop.

**Το workflow βγήκε κόκκινο (απέτυχε).**
Συνήθως είναι είτε το ΒΗΜΑ 3 (δεν δόθηκε write permission), είτε προσωρινή
συντήρηση της Διαύγειας. Πάτα στο κόκκινο ✗ για να δεις το μήνυμα, και
ξαναπροσπάθησε αργότερα με **Run workflow**.

**Πόσο κοστίζει;**
Τίποτα. Τα δημόσια/ιδιωτικά repositories, το GitHub Actions (με γενναιόδωρο
δωρεάν όριο χρόνου που δεν θα αγγίξεις) και το GitHub Pages είναι δωρεάν.

**Θέλω να αλλάξω την ώρα που τρέχει το sync.**
Άνοιξε το αρχείο `.github/workflows/sync.yml` μέσα στο GitHub (πάτα το αρχείο →
μολυβάκι ✏️), και άλλαξε τη γραμμή `cron: "0 5 * * *"`. Το `5` είναι η ώρα σε
UTC. Με το `5` τρέχει 08:00 ώρα Ελλάδας το καλοκαίρι / 07:00 τον χειμώνα.
Η ακριβής ώρα δεν έχει σημασία — αρκεί να τρέχει μία φορά τη μέρα.

---

## (Προαιρετικά) Αν θες αργότερα κανονικό κωδικό πρόσβασης

Το δωρεάν GitHub Pages δεν υποστηρίζει κωδικό. Αν κάποια στιγμή θες πραγματικό
login, μπορείς να βάλεις τη σελίδα πίσω από δωρεάν Cloudflare Access, ή να
τρέξεις την έκδοση `app.py` (που έχει ήδη κωδικό) σε δωρεάν Cloudflare Tunnel.
Πες μου και σου φτιάχνω κι αυτόν τον οδηγό.
